# CLAUDE.md

## プロジェクト概要
個人発・公開型の「雑学・知識データベースWebアプリ」。
アプリ名は TriviaVerse。
自分で調べた雑学や知識を登録・整理し、キーワードやカテゴリ（タグ）で
検索・閲覧できる知識辞典。自分は読み書きでき、他人は読むことができる。
PC・スマホ両方から同じURLで使える（公開済み）。
（フォルダ名・プロジェクト名は trivia-app）

## 解決する課題
新たな知識・雑学を増やしたいが「探す・理解する・続ける」がめんどうで定着しない。
→ 今回のスコープは「自分で調べた知識を散らからず集めて整理し、後から探せる
   状態にし、必要なら他人にも見せられる」ことに集中。
   （きっかけ作り＝再表示・通知は将来の課題として保留）

## 主要機能
- 【登録】タイトル → 本文 の順で入力して知識を追加
- 【本文2分割】AIの説明（貼り付け）と 自分のまとめ（自分の言葉）を分けて保存
  - 少なくともどちらか一方が入力されていれば登録可
- 【タグ分類】タグで分類。複数付与可・0個可・後から編集可
- 【検索・閲覧】キーワード検索＋タグ絞り込み（併用可）
- 【閲覧】詳細画面で本文・出典・タグを表示
- 【編集・削除】後から修正・削除（CRUD）
- 【権限】自分＝読み書き／他人＝読み取りのみ

## 画面構成（1ページ内で切り替える方式）
- index.html の中に、共通ヘッダー＋3つの画面エリアを持つ。
  - #view-home …… ホーム（検索・絞り込み＋登録済みの知識の一覧）
  - #view-detail … 詳細（1件をじっくり表示。ログイン中は編集・削除ボタン）
  - #view-edit …… 登録／編集フォーム
- app.js の showView("home" / "detail" / "edit") で表示を出し分ける
  （実際のページ遷移ではなく、1ページ内で見せる部分を切り替えるSPA的な方式）。
- 一覧の各カードのタイトルをクリック → 詳細画面へ。
- ログインは普段は隠し、「ログイン」ボタン(#login-toggle)を押したときだけ表示。
- 「＋ 新規登録」ボタンはログイン中のみ表示。

## データ構造（knowledgeテーブル）
- id：int8・主キー・自動採番（Is Identity）
- created_at：timestamptz・now()（自動）
- title：text（必須）
- ai_explanation：text（任意）　┐ どちらか一方は必須（アプリ側でチェック）
- my_summary：text（任意）　　 ┘
- source：text（任意）
- tags：text[]（配列・任意・複数可）

## 技術スタック
- プラットフォーム：Webアプリ（PC/スマホ両対応）
- フロントエンド：HTML / CSS / JavaScript（素のJSで開始。将来TypeScriptへ移行）
  - index.html（構造）／ app.js（処理）／ style.css（見た目）／ config.js（接続設定）
- データ保存／認証：Supabase（PostgreSQL + Auth + RLS）
  - 接続情報は config.js に記述（公開キーのみ使用）
  - ※段階3の公開時に config.js を公開対象へ変更（理由は下記「公開作業の記録」参照）
  - 秘密キー（service_role / sb_secret_）はブラウザ側で絶対に使わない
  - 注意：無料枠は7日間アクセスがないと一時停止（週1利用で回避可）
- ホスティング：Vercel（Hobbyプラン・無料）。GitHub連携で公開

## デザイン方針（段階3で実装済み）
- 雰囲気：落ち着いた知的な雰囲気（紺・グレー基調）
- 配色は style.css の :root に CSS変数でまとめ、後から調整しやすくしている
  （--navy 濃紺＝見出し/主ボタン、--base-bg グレー背景、--card-bg 白カード、
   --accent 青＝フォーカス、--border/--line グレー罫線 など）
- ボタン：主ボタン（ログイン/登録/検索）は濃紺＋白文字、副ボタンは白＋紺枠
- タグ絞り込みはチップ風（丸い形）
- 知識は白いカードで区切って表示
- スマホ対応：画面幅600px以下で余白・文字を調整し、検索欄とボタンを縦積みに

## セキュリティ設定（実施済み）
- RLS有効。ポリシー2つを設定済み：
  - 「Enable read access for all users」：SELECT / public（誰でも読み取り可）
  - 「Allow authenticated write」：ALL / authenticated（ログイン者のみ書き込み可）
- 認証：メール＋パスワード方式。自分のユーザーはSupabase管理画面で作成済み
- Data API：public スキーマ公開、knowledge テーブルを個別に公開済み
- 守っている習慣：
  - 秘密キーはコードに置かない（公開キーのみ使用）
  - 公開前に config.js の中身を必ず目視確認
  - 外部サービス連携は最小権限（Vercel の GitHub アクセスは「選択したリポジトリのみ」）
  - キー類は公開予定のものでも、画面共有時は隠す癖をつける

## 公開作業の記録（段階3・学びの備忘録）
- config.js は公開キー（sb_publishable_）のみで秘密情報を含まないため、
  .gitignore から外して公開対象にした（方式B）。
- 当初リポジトリを Private で作成 → Vercel無料(Hobby)プランでは
  Privateリポジトリのデプロイに制約があり、2回目のデプロイが Blocked になった。
- 解決：GitHubリポジトリを Public に変更 → 新しいコミットを push して再デプロイ成功。
  （config.js の中身が原因ではなく、「Private × 無料プラン」の制約が原因だった）
- GitHub認証は GitHub CLI（gh）で実施（brew install gh → gh auth login）。
- 教訓：無料で個人開発を公開するなら、最初からリポジトリを Public にしておくと
  デプロイの制約に引っかからずスムーズ（公開して困る秘密情報を置かない前提）。
- push 前に必ず git status で「何が変わるか」を確認する習慣を実践。

## 改善アイデア（バックログ／実際に使って気づいた点）
※ 2026-06-18、実際に知識を登録・閲覧して気づいた改善案。着手は今後判断。

### 案1：AIの説明と自分のまとめを視覚的に分ける（優先度：高）
- 何を：詳細・一覧で「AIの説明」と「自分のまとめ」を、色付きの枠やバッジで
  ひと目で区別できるようにする。
- なぜ：長文だとラベルだけでは境目が埋もれ、どこからが自分の言葉か分かりにくい。
- メモ：AI=グレー系の枠、自分=紺/青系の枠、など。CSS中心で実現できそう。

### 案2：書いていて楽しいデザインにする（優先度：中・要方針決め）
- 何を：機能的なだけでなく、登録・閲覧that が楽しくなる見た目・演出にする。
- なぜ：今は素朴で、書くモチベーションが上がりにくい。
- メモ：「楽しい」の方向（書くとき/見るとき/増える実感）を次回決めてから着手。

### 案3：登録数や「最新の知識」を分かりやすく（優先度：中）
- 何を：登録済みの件数を表示。ホームには最新（または今日追加した）知識を出す。
- なぜ：増えていく実感がほしい。全件が常に並ぶと、新しいものが埋もれる。
- メモ：created_at で絞り込み/並べ替え。「今日だけだと空の日がある」点に注意。
  ホームは「最新◯件＋すべて見る」など見せ分けを検討。

### 案4：過去の知識をランダムに閲覧できる画面（優先度：中）
- 何を：登録済みからランダムで1件（数件）表示する画面を追加。
- なぜ：通知ではない「能動的な再会」で、忘れかけた知識を思い出せる。
  （当初スコープ外にした「きっかけ作り」に、押し付けない形で近づく）
- メモ：4つ目の view として追加。画面切り替えの仕組みは既にあるので足しやすい。
